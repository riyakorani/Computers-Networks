# 🌐 CDN (Content Delivery Network) — DEEP DIVE

---

# 🧠 1. WHAT IS CDN?

👉 CDN is a network of distributed servers placed across different locations.

---

# 🎯 PURPOSE

- Reduce latency  
- Improve speed  
- Handle large traffic  

---

# 🌍 2. PROBLEM WITHOUT CDN

👉 User in India requests US server:

- High latency ❌  
- Slow loading ❌  

---

# 🚀 3. WITH CDN

👉 Request goes to nearest server

Example:
- User in India → gets data from India server  

✔ Faster  
✔ Lower delay  

---

# 🧪 4. HOW IT WORKS

---

## Step 1:
User requests website

---

## Step 2:
DNS routes to nearest CDN node

---

## Step 3:
CDN serves cached content

---

## Step 4:
If not cached → fetch from origin server

---

# ⚡ 5. BENEFITS

✔ Faster loading  
✔ Reduced latency  
✔ Better user experience  
✔ Handles traffic spikes  

---

# 🧠 6. REAL-WORLD EXAMPLES

- YouTube videos  
- Netflix streaming  
- Static assets (images, CSS, JS)  

---

# 🧪 SCENARIO

👉 Why YouTube loads fast globally?

👉 Because content is served from nearest CDN node

---

# 🧠 7. INTERVIEW QUESTIONS

---

## Q1: What is CDN?
👉 Distributed servers for fast delivery

---

## Q2: Why CDN improves speed?
👉 Reduces distance between user and server

---

## Q3: What is origin server?
👉 Main server where content is stored

---

## Q4: Does CDN store all data?
👉 No, mainly cached/static content

---

## Q5: What happens if CDN doesn’t have data?
👉 Fetches from origin server

---

# 🔥 FINAL SUMMARY

👉 CDN = distributed content delivery system  
👉 Reduces latency  
👉 Improves performance
